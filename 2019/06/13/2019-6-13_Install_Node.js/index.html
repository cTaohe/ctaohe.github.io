使用 Node.js

發表於 2019-06-13 | 分類於 實作

關於JavaScript開發環境的練習，認識工具，安裝 Node.js 以及相關會使用到的工具 Express、 handlerbars

2W

what?

Node.js 是什麼?

Node.js 是為 JavaScript 執行環境(run-time envirnment)，由於 JavaScript 的使用與瀏覽器是不可分割的，為了從瀏覽器解放，由美國工程師 ryan Dahl 創造，起源於 2006年。

Express 是什麼?

Express 是一種 網路框架(web framework)。讓開發者能夠在 Node.js 上面使用，快速的開發網頁應用程式。

Nodemon 是什麼?

nodemon 是用以偵測開發者專案內的檔案，如果檔案更新了，不必再使用 ctrl + c 重新啟動伺服器，可以直接更新瀏覽器，查看畫面結果。

why?

node.js 被創造起因為了解決瀏覽器上傳同步緩慢問題。

優點:

  • 能快速處理大量資料
  • 執行同時，可修正錯誤
  • 很多人使用

Express

使用 Express 的好處，經由打包好的模組，只要輸入幾個指令便可以輕鬆地將伺服器架設好，而且使用的人很多。

Nodemon

省卻重新啟動伺服器的繁瑣動作，使開發者可以專心在開發上。

Install

Node.js

node.js 從官方網 可以選擇 LTS 、 Current 版本，一個為穩定版比較不會出現 bug 適合多種環境使用，一個為較常更新的版本，使用著較新的技術，但可能會出現 bug 機會可能會多一些。

除了直接下載安裝，可以使用 terminal 透過 nvm 安裝 Node.js，以利於在不同的版本之間切換。

可參考連結文件進行實際安裝

  • macOS
  • Windows

Express

在安裝 Node.js 的時候會順帶的安裝 npm， npm 是 Node.js 的套件管理器- Node Package Manager

許多開發者為了增加效率會開發好用的 JavaScript 工具，並分享使用，為了方便搜尋使用，便有了npm 。

安裝 Express

npm i express 或 npm install express 都是一樣的意思，安裝不同的套件的時候可以透過 npm i <package> 安裝。

實際運用上，創建一個資料夾，初始化設定，在專案中建立 package.json 檔案，是為了管理套件工具，之後在裡面安裝 Express

npm init -y 中 -y 可以默認預設，減少時間在設定上。

Nodemon

安裝 Nodemon

-g 是 --global 縮寫，是在電腦做 全域安裝 安裝過後一次便不用再重新下載安裝。

使用 nodemon 代替 node

接下來就可以輕鬆使用 nodemon 愉快的測試專案了

# express # nodemon # Node.js
